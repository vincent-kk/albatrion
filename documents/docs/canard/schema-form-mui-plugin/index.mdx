## Installation

```bash
yarn add @canard/schema-form-mui-plugin
```

### Peer Dependencies

```bash
yarn add @mui/material@>=7.0.0 @mui/x-date-pickers@>=8.0.0 \
         @mui/icons-material@>=7.0.0 \
         @emotion/react@>=11.0.0 @emotion/styled@>=11.0.0 \
         dayjs@>=1 react@>=18 react-dom@>=18
```

## Quick Setup

```ts
import { registerPlugin } from '@canard/schema-form';
import { plugin } from '@canard/schema-form-mui-plugin';

registerPlugin(plugin);
```

## Component Mapping

Components are selected automatically based on JSON Schema properties. Priority order matches the list below (top = highest priority).

| Component | type | format | formType | enum | Notes |
|---|---|---|---|---|---|
| `FormTypeInputBooleanSwitch` | `boolean` | ‚Äî | `"switch"` | ‚Äî | MUI Switch |
| `FormTypeInputStringCheckbox` | `array` | ‚Äî | `"checkbox"` | items.enum | MUI Checkbox group; items must be string schema |
| `FormTypeInputStringSwitch` | `string` | ‚Äî | `"switch"` | 2 values | Toggle between two enum values |
| `FormTypeInputUri` | `string` | `"uri"` or formType `"uri"` | ‚Äî | ‚Äî | URI input with protocol dropdown |
| `FormTypeInputMonth` | `string` | `"month"` | ‚Äî | ‚Äî | MUI X MonthPicker |
| `FormTypeInputDate` | `string` | `"date"` | ‚Äî | ‚Äî | MUI X DatePicker |
| `FormTypeInputTime` | `string` | `"time"` | ‚Äî | ‚Äî | MUI X TimePicker (HH:mm:00Z format) |
| `FormTypeInputRadioGroup` | `string` \| `number` \| `integer` | ‚Äî | `"radio"` or `"radiogroup"` | required | MUI RadioGroup |
| `FormTypeInputStringEnum` | `string` | ‚Äî | ‚Äî | required | MUI Select dropdown |
| `FormTypeInputArray` | `array` | ‚Äî | ‚Äî | ‚Äî | Dynamic list with add/remove |
| `FormTypeInputSlider` | `number` \| `integer` | ‚Äî | `"slider"` | ‚Äî | MUI Slider |
| `FormTypeInputTextarea` | `string` | `"textarea"` or formType `"textarea"` | ‚Äî | ‚Äî | MUI TextField multiline |
| `FormTypeInputString` | `string` | ‚Äî | ‚Äî | ‚Äî | MUI TextField (password via `format:"password"`) |
| `FormTypeInputNumber` | `number` \| `integer` | ‚Äî | ‚Äî | ‚Äî | MUI TextField type=number |
| `FormTypeInputBoolean` | `boolean` | ‚Äî | ‚Äî | ‚Äî | MUI Checkbox with indeterminate |

## Schema Extension Properties

```ts
{
  formType?: 'switch' | 'radio' | 'radiogroup' | 'checkbox' | 'slider' | 'textarea' | 'uri',
  format?:   'date' | 'time' | 'month' | 'password' | 'uri' | 'textarea',
}
```

## Usage Example

```tsx
import { Form } from '@canard/schema-form';

const schema = {
  type: 'object',
  properties: {
    name:     { type: 'string' },
    age:      { type: 'integer' },
    active:   { type: 'boolean', formType: 'switch' },
    role:     { type: 'string', enum: ['admin', 'user', 'guest'] },
    birthday: { type: 'string', format: 'date' },
    bio:      { type: 'string', format: 'textarea' },
    score:    { type: 'number', formType: 'slider', minimum: 0, maximum: 100 },
  },
};

<Form jsonSchema={schema} onChange={setValue} />
```

## MUI Context

Pass MUI styling options through `<Form context={...}>`:

```ts
type MuiContext = {
  size?:      'small' | 'medium';
  variant?:   'outlined' | 'filled' | 'standard';
  fullWidth?: boolean;
};
```

```tsx
<Form
  jsonSchema={schema}
  context={{ size: 'small', variant: 'outlined', fullWidth: true }}
  onChange={setValue}
/>
```

## Date/Time Picker Setup

MUI X date pickers require a `LocalizationProvider`. Wrap your app:

```tsx
import { LocalizationProvider } from '@mui/x-date-pickers';
import { AdapterDayjs } from '@mui/x-date-pickers/AdapterDayjs';

<LocalizationProvider dateAdapter={AdapterDayjs}>
  <Form jsonSchema={schema} onChange={setValue} />
</LocalizationProvider>
```
