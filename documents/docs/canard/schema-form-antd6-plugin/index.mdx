## Installation

```bash
yarn add @canard/schema-form-antd6-plugin
```

### Peer Dependencies

```bash
yarn add antd@>=6.0.0 dayjs@>=1 react@>=18 react-dom@>=18
```

## Quick Setup

```ts
import { registerPlugin } from '@canard/schema-form';
import { plugin } from '@canard/schema-form-antd6-plugin';

registerPlugin(plugin);
```

## Component Mapping

Components are selected automatically based on JSON Schema properties. Priority order matches the list below (top = highest priority).

| Component | type | format | formType | enum | Notes |
|---|---|---|---|---|---|
| `FormTypeInputBooleanSwitch` | `boolean` | ‚Äî | `"switch"` | ‚Äî | Switch toggle |
| `FormTypeInputStringCheckbox` | `array` | ‚Äî | `"checkbox"` | items.enum | Checkbox group; items must be string schema |
| `FormTypeInputStringSwitch` | `string` | ‚Äî | `"switch"` | 2 values | Toggle between two enum values |
| `FormTypeInputUri` | `string` | `"uri"` or formType `"uri"` | ‚Äî | ‚Äî | URI input with protocol selector |
| `FormTypeInputMonth` | `string` | `"month"` | ‚Äî | ‚Äî | Month picker |
| `FormTypeInputDate` | `string` | `"date"` | ‚Äî | ‚Äî | Date picker |
| `FormTypeInputTime` | `string` | `"time"` | ‚Äî | ‚Äî | Time picker |
| `FormTypeInputMonthRange` | `array` | `"month-range"` or formType `"monthRange"` | ‚Äî | ‚Äî | Month range picker; items must be string schema |
| `FormTypeInputDateRange` | `array` | `"date-range"` or formType `"dateRange"` | ‚Äî | ‚Äî | Date range picker; items must be string schema |
| `FormTypeInputTimeRange` | `array` | `"time-range"` or formType `"timeRange"` | ‚Äî | ‚Äî | Time range picker; items must be string schema |
| `FormTypeInputRadioGroup` | `string` \| `number` \| `integer` | ‚Äî | `"radio"` or `"radiogroup"` | required | Radio button group |
| `FormTypeInputStringEnum` | `string` | ‚Äî | ‚Äî | required | Select dropdown; also handles string-item arrays with enum |
| `FormTypeInputArray` | `array` | ‚Äî | ‚Äî | ‚Äî | Dynamic list with add/remove |
| `FormTypeInputSlider` | `number` \| `integer` | ‚Äî | `"slider"` | ‚Äî | Slider input |
| `FormTypeInputTextarea` | `string` | `"textarea"` or formType `"textarea"` | ‚Äî | ‚Äî | Multi-line textarea |
| `FormTypeInputString` | `string` | ‚Äî | ‚Äî | ‚Äî | Default text input (password supported via `format: "password"`) |
| `FormTypeInputNumber` | `number` \| `integer` | ‚Äî | ‚Äî | ‚Äî | Numeric input |
| `FormTypeInputBoolean` | `boolean` | ‚Äî | ‚Äî | ‚Äî | Checkbox |

## Schema Extension Properties

```ts
// Custom schema properties recognized by this plugin
{
  formType?: 'switch' | 'radio' | 'radiogroup' | 'checkbox' | 'slider' |
             'textarea' | 'uri' | 'dateRange' | 'monthRange' | 'timeRange',
  // Standard JSON Schema properties also used for routing:
  format?: 'date' | 'time' | 'month' | 'password' | 'uri' |
           'textarea' | 'date-range' | 'month-range' | 'time-range',
}
```

## Usage Example

```tsx
import { Form } from '@canard/schema-form';

const schema = {
  type: 'object',
  properties: {
    name:      { type: 'string' },
    age:       { type: 'integer' },
    active:    { type: 'boolean', formType: 'switch' },
    role:      { type: 'string', enum: ['admin', 'user', 'guest'] },
    birthday:  { type: 'string', format: 'date' },
    bio:       { type: 'string', format: 'textarea' },
    score:     { type: 'number', formType: 'slider', minimum: 0, maximum: 100 },
  },
};

<Form jsonSchema={schema} onChange={setValue} />
```

## Context

The plugin reads `context` passed to `<Form context={...}>` for size configuration:

```ts
type AntdContext = {
  size?: 'small' | 'middle' | 'large';
};
```

## v5 vs v6

The antd5 and antd6 plugins are API-identical. The only difference is the `antd` peer dependency version. Switch packages by replacing the import and peer dependency:

```ts
// antd5
import { plugin } from '@canard/schema-form-antd5-plugin';
// antd6
import { plugin } from '@canard/schema-form-antd6-plugin';
```
