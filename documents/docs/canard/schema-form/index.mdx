JSON Schema-driven React form engine with a two-layer plugin architecture:
a **Validator plugin** for JSON Schema validation and
a **UI plugin** for form rendering components.
Declare your data shape once in JSON Schema — the library builds, validates,
and manages the full form state tree automatically.

## Architecture

```
JSON Schema
    │
    ▼
nodeFromJsonSchema()          ← builds a typed node tree
    │
    ├── StringNode / NumberNode / BooleanNode / NullNode   (terminal)
    ├── ObjectNode / ArrayNode                             (branch)
    └── VirtualNode                                        (conditional / computed)
    │
    ▼
Plugin System
    ├── ValidatorPlugin       ← compile(jsonSchema) → ValidateFunction
    └── SchemaFormPlugin      ← FormGroup / FormLabel / FormInput / FormError
                                 + formTypeInputDefinitions
    │
    ▼
<Form>  →  React component tree
```

## Plugin System

The library ships with *no default validator and no default UI components*.
Both must be supplied via plugins before rendering any form.
Plugins are registered globally with `registerPlugin()` and affect every
`<Form>` in the application.

<ul>
  <li><strong>Validator plugin</strong> — integrates any validation library (AJV, Zod, Yup, etc.) by implementing <code>compile(jsonSchema)</code>.</li>
  <li><strong>UI plugin</strong> — provides <code>FormGroup</code>, <code>FormLabel</code>, <code>FormInput</code>, <code>FormError</code> renderer components and <code>formTypeInputDefinitions</code> for per-field input components.</li>
</ul>

Official plugin packages:

<ul>
  <li><code>@canard/schema-form-ajv8-plugin</code> — AJV 8 validator</li>
  <li><code>@canard/schema-form-antd5-plugin</code> — Ant Design 5 UI</li>
  <li><code>@canard/schema-form-mui6-plugin</code> — MUI 6 UI</li>
</ul>

## Node Types

| Node | Schema types | Strategy |
|---|---|---|
| `StringNode` | `"string"` | terminal |
| `NumberNode` | `"number"`, `"integer"` | terminal |
| `BooleanNode` | `"boolean"` | terminal |
| `NullNode` | `"null"` | terminal |
| `ObjectNode` | `"object"` | branch (BranchStrategy or TerminalStrategy) |
| `ArrayNode` | `"array"` | branch (BranchStrategy or TerminalStrategy) |
| `VirtualNode` | conditional / oneOf / anyOf branches | special |

## FormTypeInput Resolution Order

When rendering a field, the library resolves which input component to use in this priority order (highest first):
<ol>
  <li>Direct <code>FormTypeInput</code> prop on <code>Form.Group</code></li>
  <li><code>formTypeInputMap</code> path mapping on <code>&lt;Form&gt;</code></li>
  <li>Form-level <code>formTypeInputDefinitions</code> prop</li>
  <li><code>FormProvider</code> <code>formTypeInputDefinitions</code></li>
  <li>Plugin-provided <code>formTypeInputDefinitions</code></li>
</ol>

## Claude Code Integration

`@canard/schema-form` ships Claude Code assets — the `schema-form-skill` expert skill, its knowledge base, and the `schema-form-rule` authoring rule — under `docs/claude/`. Each release publishes them alongside the package so AI assistants can reason about the form engine with package-specific context.

Inject those assets into your local Claude Code environment with the bundled `claude-sync` CLI (engine: `@slats/claude-assets-sync`):

```bash
# Direct dependency — bin is hoisted to your project's .bin
npx claude-sync --scope=user         # inject into ~/.claude (user-level)
npx claude-sync --scope=project      # inject into the nearest existing .claude, walking up from cwd

# Preview / overwrite flags
npx claude-sync --scope=user --dry-run   # show the plan without writing
npx claude-sync --scope=user --force     # overwrite locally-modified files

# Transitive dependency — pin the package explicitly
npx -p @canard/schema-form claude-sync --scope=user
```

The CLI is a thin stub: it reads `@canard/schema-form`'s own `package.json`, resolves `claude.assetPath` (`docs/claude`), and copies the tree into the selected `.claude` directory with hash-based change detection (skip unchanged, warn on divergence, require `--force` to overwrite).
