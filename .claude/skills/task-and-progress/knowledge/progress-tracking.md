### 자동 생성 로직 (tools/progress_tracker.sh)

**입력**:
- 03_plan.md (체크박스 상태)
- 이전 progress_log.md (누적 데이터)

**출력**:
- 업데이트된 progress_log.md
- 진행률 통계

**계산**:
```bash
# 완료 작업 수
completed=$(grep -c "\[x\]" 03_plan.md)

# 전체 작업 수
total=$(grep -c "^###.*" 03_plan.md)

# 진행률
progress=$((completed * 100 / total))

# Velocity (작업/일)
days_elapsed=$(($(date +%s) - $(date -d "$start_date" +%s)))
days=$((days_elapsed / 86400 + 1))
velocity=$(echo "scale=1; $completed / $days" | bc)

# 예상 완료일
remaining=$((total - completed))
days_remaining=$(echo "scale=0; $remaining / $velocity" | bc)
estimated_end=$(date -d "+${days_remaining} days" +%Y-%m-%d)
```

## Velocity 계산

### Velocity란?

**정의**: 단위 시간당 완료한 작업 수

```
Velocity = 완료한 작업 수 / 경과 일수

예:
- Day 1: 2 작업 완료 → Velocity = 2
- Day 2: 누적 5 작업 완료 → Velocity = 5/2 = 2.5
- Day 3: 누적 7 작업 완료 → Velocity = 7/3 = 2.33
```

### Velocity 활용

**예상 완료일 계산**:
```
남은 작업 수 / Velocity = 남은 일수

예:
- 전체: 17 작업
- 완료: 7 작업
- 남은: 10 작업
- Velocity: 2.33 작업/일
- 남은 일수: 10 / 2.33 = 4.29일 ≈ 5일
```

**일정 위험 탐지**:
```
예상 완료일 > 목표 완료일
→ 일정 지연 위험

예:
- 목표 완료: 2024-01-20
- 예상 완료: 2024-01-24
- 위험: 4일 지연 예상
→ 대응: 작업 우선순위 재조정, 리소스 추가
```

## 블로커 및 이슈 추적

### 블로커 (Blocker)

**정의**: 작업 진행을 완전히 막는 문제

```markdown
### 블로커 🚨
1. **API 서버 다운**
   - 발견: 2024-01-16 14:00
   - 영향: 1.4, 1.5 작업 불가
   - 담당: Backend 팀
   - 상태: 해결 대기 중
   - 예상 해결: 2024-01-16 18:00

2. **라이브러리 버그**
   - 발견: 2024-01-17 10:00
   - 영향: 2.5 드래그 선택 작업
   - 우회 방안: 다른 라이브러리 검토 중
   - 상태: 진행 중
```

### 이슈 (Issue)

**정의**: 해결 가능하지만 주의 필요한 문제

```markdown
### 이슈 ⚠️
1. **성능 저하**
   - 현상: 100개+ 아이템 렌더링 시 600ms 소요
   - 목표: 500ms 이하 (REQ-NF-001)
   - 해결 방안: 가상 스크롤 적용 예정
   - 우선순위: Medium
   - 담당: {이름}

2. **타입 불일치**
   - 현상: API 응답 타입이 문서와 다름
   - 영향: 1.3 작업에서 타입 에러
   - 해결: 타입 정의 수정 완료
   - 상태: 해결됨 ✅
```
