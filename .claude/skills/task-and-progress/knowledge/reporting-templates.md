## 주요 이슈 및 해결

### Issue 1: 드래그 선택 모바일 미지원
- **발견**: Day 3, 작업 2.5 진행 중
- **문제**: react-dnd 라이브러리가 터치 이벤트 미지원
- **영향**: REQ-F-011 (Medium 우선순위) 미달성 위험
- **해결**:
  1. react-dnd-touch-backend 추가 설치
  2. MultiBackend 설정으로 마우스/터치 동시 지원
  3. 추가 시간: 2시간
- **결과**: ✅ 해결됨, 요구사항 100% 충족
- **학습**: 모바일 지원이 필요한 기능은 초기에 라이브러리 검증 필요

### Issue 2: 성능 목표 미달 (초기)
- **발견**: Day 4, 작업 3.2 성능 테스트
- **문제**: 100개 아이템 렌더링 시 600ms 소요 (목표: 500ms)
- **영향**: REQ-NF-001 미달성
- **해결**:
  1. react-window 가상 스크롤 적용
  2. TimeSlotItem React.memo 최적화
  3. 불필요한 리렌더링 제거
  4. 최종 성능: 450ms
- **결과**: ✅ 해결됨, 목표 대비 110% 달성
- **학습**: 대량 리스트는 초기부터 가상 스크롤 고려

### Issue 3: API 응답 타입 불일치
- **발견**: Day 2, 작업 1.4 API 클라이언트 구현
- **문제**: 실제 API 응답이 문서와 다름
- **영향**: 타입 에러 발생, 작업 지연 0.5시간
- **해결**:
  1. Backend 팀에 확인
  2. 실제 응답 기준으로 타입 수정
  3. API 문서 업데이트 요청
- **결과**: ✅ 해결됨
- **학습**: API 통합 전 실제 응답 확인 필요

## 학습 내용 (Lessons Learned)

### 기술적 학습
1. **Jotai Atom Composition**
   - 학습: atom(get => ...) 패턴으로 파생 상태 선언적 관리
   - 효과: 불필요한 리렌더링 70% 감소
   - 활용: 향후 모든 React 프로젝트에 Jotai 적용 고려

2. **react-window 가상 스크롤**
   - 학습: FixedSizeList로 대량 리스트 렌더링 최적화
   - 효과: 100개 아이템 렌더링 600ms → 450ms (25% 개선)
   - 활용: 50개 이상 리스트는 기본적으로 가상 스크롤 적용

3. **ARIA 접근성**
   - 학습: role, aria-checked, aria-label로 스크린 리더 지원
   - 효과: WCAG 2.1 AA 100% 준수
   - 활용: 접근성 테스트를 CI/CD에 통합 (axe-core)

### 프로세스 학습
1. **모바일 우선 검증**
   - 문제: 데스크톱에서만 테스트하여 모바일 이슈 늦게 발견
   - 개선: 초기부터 모바일 테스트 병행
   - 적용: 다음 프로젝트에서는 모바일 우선 개발

2. **성능 목표 초기 설정**
   - 문제: Phase 3에서야 성능 테스트 → 수정 비용 증가
   - 개선: Phase 1부터 성능 측정 도구 설정
   - 적용: 성능 목표는 초기 작업에 포함

3. **API 문서 검증**
   - 문제: 문서와 실제 API 불일치로 시간 낭비
   - 개선: API 통합 전 실제 응답 확인
   - 적용: API 문서 자동 생성 도구 도입 검토
