# claude-assets-sync Specification

## Requirements

- Consumer packages expose an `inject-docs` bin that copies `docs/claude/**` into the user's chosen scope (`user`/`project`/`local`)
- File-level SHA-256 comparison: copy on absence, skip on match, warn-and-require-`--force` on mismatch (Option A: user-edit vs source-update indistinguishable)
- `--force` in TTY: interactive confirm prompt showing up to 3 diverged file names + "in git?" question; in non-TTY: emit diverged/orphan list to stderr and proceed
- `--dry-run`: print plan without writing
- Non-TTY + missing `--scope`: exit 2 with explicit error
- Legacy 0.1.x commands (`sync`, `add`, `list`, `remove`, `status`, `migrate`, `update`) emit deprecation stub (exit 1, point to MIGRATION.md)
- `dist/claude-hashes.json` generated by `buildHashes` at build time; schema v1 with `previousVersions: {}` reserved for future Option A+

## API Contracts

- `program(options: ProgramOptions): Promise<void>` — `./cli` subpath. Options: `packageName`, `packageVersion`, `packageRoot`, optional `assetRoot`, `argv`
- `injectDocs(opts: InjectOptions): Promise<InjectReport>` — programmatic. Exit codes: 0 success/up-to-date/dry-run, 1 runtime error, 2 user/config error
- `readHashManifest(packageRoot): Promise<HashManifest>` — reads `packageRoot/dist/claude-hashes.json`
- `computeNamespacePrefixes(manifest): string[]` — derives `skills/<name>/` prefixes for orphan scoping
- `resolveScope(scope, cwd?): ScopeResolution`
- `buildHashes(opts?): Promise<{ outPath, fileCount }>` — `./buildHashes` subpath; self-executing CLI via `claude-build-hashes` bin. Ignores `.omc/**`, `.DS_Store`, `*.log`
