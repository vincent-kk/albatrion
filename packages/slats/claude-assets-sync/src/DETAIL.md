# claude-assets-sync Specification

## Requirements

- Consumer packages expose a `claude-sync` bin (3-line re-export stub) that discovers their sibling packages via `claude.assetPath` in `package.json` and injects `docs/claude/**` into the user's chosen scope (`user`/`project`/`local`)
- `claude-sync` also accepts `list` and `build-hashes` subcommands; legacy `inject-docs` alias stays functional
- File-level SHA-256 comparison: copy on absence, skip on match, warn-and-require-`--force` on mismatch (Option A: user-edit vs source-update indistinguishable)
- `--force` in TTY: interactive confirm via `@inquirer/prompts.confirm` showing up to 3 diverged/orphan file names; non-TTY: emit diverged/orphan list to stderr and proceed
- `--dry-run`: print plan without writing
- Non-TTY + missing `--scope`: exit 2 with explicit error
- Legacy 0.1.x commands (`sync`, `add`, `list`-legacy, `remove`, `status`, `migrate`, `update`) emit deprecation stub (exit 1, point to MIGRATION.md)
- `dist/claude-hashes.json` generated by `buildHashes` at build time; schema v1 with `previousVersions: {}` reserved for future Option A+

## API Contracts

- `runCli(argv, options?): Promise<void>` — primary entry. Options: `{ version? }`. Drives discover → inject / list / build-hashes.
- `discover(options?): Promise<ConsumerPackage[]>` — walks parent workspaces and every ancestor `node_modules`; returns packages with `claude.assetPath` declared. Options: `{ cwd?, includeWorkspaces? }`.
- `injectDocs(opts: InjectOptions): Promise<InjectReport>` — headless programmatic inject. Exit codes: 0 success/up-to-date/dry-run, 1 runtime error, 2 user/config error.
- `program(options: ProgramOptions): Promise<void>` — `@deprecated` legacy factory bound to the old `inject-docs` bin; retained as forwarding shim.
- `readHashManifest(packageRoot): Promise<HashManifest>` — reads `packageRoot/dist/claude-hashes.json`
- `computeNamespacePrefixes(manifest): string[]` — derives `skills/<name>/` prefixes for orphan scoping
- `resolveScope(scope, cwd?): ScopeResolution`
- `buildHashes(opts?): Promise<{ outPath, fileCount }>` — `./buildHashes` subpath (Node ESM); self-executing CLI via `claude-build-hashes` bin. Ignores `.omc/**`, `.DS_Store`, `*.log`.

## Exported Types

- `RunCliOptions`, `ConsumerPackage`, `DiscoverOptions`
- `ProgramOptions`, `InjectOptions`, `InjectReport`
- `HashManifest`, `Scope`, `ScopeResolution`, `ClaudeConfig`, `PackageInfo`, `AssetType`
